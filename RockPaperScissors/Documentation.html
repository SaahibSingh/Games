Rock Paper Scissors – Project Overview

This repository contains a simple console-based implementation of the classic Rock, Paper, Scissors game in Java. The project demonstrates:

  • Basic game logic and control flow
  • Use of classes, static and instance variables
  • Random number generation
  • User input handling via the console

How the game works

The game is played in rounds between the user and the computer. Before each game starts, the user chooses how many rounds are needed to win the game (for example, first to 3 wins).

  1. Choose rounds to win: The user enters a positive integer, or 0 to quit.
  2. Play rounds: In each round, the user enters rock, paper, or scissors. The computer randomly picks one of the three.
  3. Determine the round winner:
    • Rock beats scissors
    • Scissors beats paper
    • Paper beats rock
    • Same choice results in a tie
  4. End of game: When either the user or the computer reaches the target number of round wins, the game announces the winner.
  5. Play again: The user can choose to start a new game, repeating the process.
  6. Overall statistics: When the user quits, overall statistics for all games played in this run are displayed.

Project structure

All classes are in the rockpaperscissors package:

Randomizer.java

Utility Class

A small utility class that wraps Math.random() to generate random integers. It provides reusable methods to get random numbers in a range.

  • public static int nextInt()
    Returns a random integer between 1 and 10 (inclusive).
  • public static int nextInt(int min, int max)
    Returns a random integer in the range [min, max] (inclusive). This is used by the game to pick the computer's move (rock, paper, or scissors).
Note: The game uses nextInt(0, 2) to map 0 → rock, 1 → paper, 2 → scissors.

RockPaperScissors.java

Core Game Logic

This class represents a single game session and also tracks overall statistics across all games played while the program is running.

Class (static) fields

  • totalGamesPlayed – total number of game sessions started.
  • totalUserGameWins – total games won by the user.
  • totalComputerGameWins – total games won by the computer.
  • totalGameTies – total games ending in a tie.

Instance fields (per game session)

  • userRoundWins – number of rounds the user has won in this game.
  • computerRoundWins – number of rounds the computer has won in this game.
  • roundsNeededToWin – target number of round wins needed to win the game.
  • ties – number of tied rounds in this game.

Key methods

  • public RockPaperScissors(int roundsToWin)
    Initializes a new game with the required number of rounds to win, resets per-game counters, and increments totalGamesPlayed.
  • public static String getRandomChoice()
    Uses Randomizer to generate an integer between 0 and 2 and converts it to "rock", "paper", or "scissors".
  • public static void printOverallStats()
    Prints the accumulated statistics for all games played: total games, user wins, computer wins, and ties.
  • public String determineRoundWinner(String user, String computer)
    Compares the user’s choice and computer’s choice and returns: "user", "computer", or "tie".
  • public String playRound(String userChoice)
    Generates a computer choice, determines the round winner, updates the round counters, and returns a human-readable message like "User wins this round!".
  • public boolean hasGameWinner()
    Returns true if either the user or the computer has reached roundsNeededToWin; otherwise false.
  • public String getGameWinnerMessage()
    Determines who won the game based on round wins, updates the appropriate static counters (totalUserGameWins, totalComputerGameWins, or totalGameTies), and returns a message announcing the winner.
  • public void printCurrentGameScore()
    Prints the current number of user wins, computer wins, and ties for this game session.

RPSGameRunner.java

Program Entry Point

This class contains the main method and is responsible for user interaction and controlling the game loop.

Responsibilities

  • Prompt the user for the number of rounds needed to win.
  • Create a RockPaperScissors game instance.
  • Read and validate the user’s rock/paper/scissors input each round.
  • Call playRound and display round results and current score.
  • Check hasGameWinner() to know when the game ends.
  • Ask the user if they want to play another game.
  • On exit, close the Scanner, print overall stats, and say goodbye.

Running the program

Make sure your files are in a folder structure that matches the package: 

src/
  rockpaperscissors/
    Randomizer.java
    RockPaperScissors.java
    RPSGameRunner.java

From the root of the project, compile and run:

javac src/rockpaperscissors/*.java
java -cp src rockpaperscissors.RPSGameRunner

Game flow summary

  1. User chooses how many rounds are needed to win.
  2. Rounds are played until user or computer reaches that number of wins.
  3. Each round:
    • User enters rock, paper, or scissors.
    • Computer randomly chooses.
    • Round result is displayed and scores are updated.
  4. When the game ends, the winner is announced.
  5. User decides whether to start a new game.
  6. When user quits, overall statistics across all games are displayed.
